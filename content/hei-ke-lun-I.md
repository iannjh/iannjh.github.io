# 输出构成基本问题

### 逻辑形式

如果非要从头说起，有效输出构成，即战斗机制的底层逻辑必然取决于事件基础（EventManager），如果没有事件基础对每个符号量的判定，无论是硬符号还是软符号，都无法被实现，事件（Event）决定了这些或者那些量**如何**引发质变反应，很显然，恩格斯早就在自然辩证法里论证了这个逻辑，亦或者说，没有自然辩证法的指导，我们无法研究 Minecraft 战斗机制，恩格斯在《自然辩证法》中明确指出：

> 在同系列的碳化物、特别是较简单的碳氢化合物中，这一点表现得更为显著。在正烷烃中，最低级的是甲烷，$\mathrm{CH_4}$；在这里，碳原子的四个化学键被四个氢原子所饱和。第二种是乙烷，$\mathrm{C_2H_6}$，两个碳原子互相联结，自由的六个化学键被六个氢原子所饱和。以下依据代数学的公式 $\mathrm{C_\mathit{n}H_{2\mathit{n}＋2}}$ ，便有 $\mathrm{C_3H_8}$ ，$\mathrm{C4H10}$ 等等，所以每增加一个 $\mathrm{CH}_2$，便形成一个和以前的物体在质上不同的物体。这一系列中最低的三种是气体，已知的最高的一种十六烷，$\mathrm{C_{16}H_34}$，是沸点为摄氏270度的一种固体。关于从烷烃（理论上）得出的伯醇系列（公式是$\mathrm{C_\mathit{n}H_{2\mathit{n}+2}O}$）和一元脂肪酸系列（公式为$\mathrm{C_\mathit{n}H_{2\mathit{n}}O_2}$），情形也完全一样。在量上加上一个C3H6，能够造成什么样的质的区别，可以从如下的经验看出来：我们喝可以饮用的并且不掺杂其他醇类的乙醇$\mathrm{C_2H_6O}$，另一次我们喝同样的乙醇，但掺入了小量的戊醇 $\mathrm{C_5H_{12}O}$（它是大名鼎鼎的杂醇油的主要成分）。第二天早晨我们的脑袋就一定会感到这个东西，而且觉得受到它的伤害；所以甚至可以说：醉酒和由之而来的醉后头痛正是量到质的转化，一方面是乙醇，另一方面是这一点加上去的 $\mathrm{C_3H_6}$ 。

在 Java 中，常量通常在程序运行过程中不会改变其值，Java 中定义常量的格式是 `（修饰符）final 数据类型 常量名 = 变量值;`，其中只有修饰符是可选的，其他都是必要项，例如`public final int WXX = 1`。而变量则在程序运行中可以改变其值，Java中其定义的格式为`（修饰符）数据类型 变量名 =（变量值）`，如果是全局变量，变量值可以不赋值（`int x;`），如果是局部变量必须初始化（`int x; x = 1;`），由此可见，Java及任何编程语言的基础，都是开发人员对因变量和自变量互相调整的结果，因变量的出现是为了避免自变量的矛盾，对自变量的问题进行界定，无论方法如何，自变量作为变化的原因，因变量作为变化的条件，实现是它们之间相互作用的反馈。

因此事件基础是Java之本，而对事件结果的管理，也说对反馈机制的管理，由于其复杂性造成了不同反馈之间不能仅仅依赖交互，也就是说，如果要在交互的基础上开发新的变更或让程序开发显得更直观，就需要引入新的模块管理系统，否则，交互间的关系就会被复杂的更改破坏。通常，一个程序提供逻辑要能够满足新的开发者的基础需求，即

- a. 能够管理和创造事件
- b. 能够使得事件之间不发生矛盾
- c. 能够让开发者在不破坏事件基础的情况下尽可能创造新的功能，

便能成为一个完美的程序，但这根本不简单——矛盾对立的统一不仅要解决本体与外部的矛盾，本体自身能够实现本身就是矛盾的结果而不是原因，因此内因是变化的条件，而外因是变化的环境，环境本身也是不同内因互相交互产生的结果，内因终究还是决定外因，这就是矛盾的本质。

方法论的诞生是**本体论无法承载不同的质之间抽象判别从而产生的结果**，这就要谈到映射（Map），从映射的角度而言，这种传递相比较于交互和管理更加直接，而由于映射的对象是物理的输入，因而对于编程语言的要求更高，但这也造就了它需要编程语言需要大量的依赖库和类库来完成它与物理的接触，这看似很简单，但映射对性能的损耗和管理的准确性至关重要，如果依赖有一点瑕疵，程序不但要面临安全风险，甚至可能面临崩溃或无法运行，编译性语言在这个时候已经无法承载物理与程序之间反馈的复杂性，因此解释性语言就诞生了——它试图通过脚本和解释器对我结合来减小反馈的复杂性和依赖库的局限性，它更好的打破了物理映射带来的困难和冲突，它的库显得更加集中，而不是更加分散。

### 底层逻辑关系

有效输出构成（logic src constitute，简称LSC），即战斗机制的底层逻辑，在pvp模式中表现为硬符号的常量，也就是狭义上的有效输出构成符号，但在游戏源代码的语境下就变成了广义有效输出构成，它们的共同性质是都与资源逻辑和形式逻辑的抽象输出构成存在对立，这两者在性质上是完全不相干的。

行为主义者则完全不同，他们把抽象输出构成作为事件的前提，从而让大多数PVP玩家达成一种先验的观念，也就是说

- a. 资源在未加载和配置完成之前，游戏不存在任何逻辑
- b. 逻辑是符号的结果，而不是原因。
- c. 底层逻辑不能脱离物理条件和节制度独立存在。

这就是说，他们把硬符号看作对游戏逻辑的直接修改或破坏，而不是继承，整体上，他们认同玩家通过经验积累提高技术和游戏体验，另一方面，他们又否认了有效输出构成和抽象输出构成存在不可调和的隔阂。

行为主义者在叙事上将我的世界PVP模式的诞生看作是从PVP行为演变过来的，但忘记了，**PVP模式本身就是游戏事件和资源沟通的必要结果，它甚至不能作为独立的模式存在**，如果玩家不通过特定手段改变资源构成的结构，那他们就无法进行游戏，同时，行为主义者忽略了游戏战斗机制普遍存在的经济问题，将PVP模式的成败完全归结于经验，这种小农式的思维导致了玩家常常对游戏的不当竞争事件产生怀疑，形成了众说纷纭的阴谋论和怀疑论哲学（例如所谓的 misaim 和 misplaced 焦虑病），如果不打破用行为主义方法研究战斗机制问题，那就算 Minecraft 创新和纠错的频率再高，它也无法造福玩家。

研究抽象输出构成本身没有意义，或者说，它是完全违背 Minecraft 战斗机制的运作逻辑的，行为主义本身就是一种没有本体论的方法论，行为主义是一种纯粹主义，将 Minecraft 的硬符号和软符号视为没有区别，但这两者是天差地别的，试图打破它们的界限，那我的世界将不复存在，抽象输出构成塑造的软符号除了造成经验焦虑之外，也无法指导实践，行为主义者甚至根本没有意识到我的世界不是基于物理引擎的游戏，而是基于复杂的逻辑构成的软件，脱离本体论就夸夸其谈方法论，本质上是对科学的蔑视——马克思利用辩证唯物主义向世人揭示了资本主义经济学对利润率的粗暴解读，我们要做的便是把专有软件学者对构成本体论的粗暴解读彻底批判，维护辩证唯物主义的精髓。

如果非要谈目的论，那么行为主义战斗机制学和结构主义战斗机制学在方法论上是完全统一的，是以行为主义为手段达成经验主义的目的，后者则是用结构主义手段达成解构主义的目的，**实现则是他们的终结**，无论双方用何种方式指控对方，在目的实现之前，道路都是迷茫的，专有商业公司并非一开始就战胜了开源，科学最初是开源的，因此开源是从开源走向开源，自由软件主义本身就是科学理论，它不依赖意志的转移和需求的变化，反观行为主义，一直在重蹈覆辙，而这种循环，必然是要被按下暂停键的。

### 作为专有软件的我的世界

但从ms和mojang领导视角来说，我的世界本身的效用往往表现为随资源逻辑和底层逻辑交互的变化而下降，但玩家在此过程中只需要透过资源逻辑完成对形式逻辑的理解，而这种主观的理解不足以使玩家掌握行为规律的变动，而微软专有化领导下，无政府服务器和反作弊服务器的对立自然不可调节，于是，它们形成了市场关系，但mojang和ms对它们的干预极低，造成了这种市场竞争让社区内部产生极大困扰，无政府服务器对外的市场关系和对内玩家客户端的技术垄断和对公会组织秘密的封锁仍然无法改变这种意识形态的劣根性，本质上，新pvp主义并不关心玩家自由和行为规律的覆灭，而是更在乎其模式的优越性能否战胜反作弊主义服务器，但这是根本不可能的，一旦利益集团形成，反作弊服务器服主必然倒向贿赂的公会，并且让该公会建立新的潜在秩序，这种秩序使得它们对服务器进行了专有化封锁，服务器团队仅负责营销，而服务器内部则要求玩家必须和公会组织搞好关系，渐渐地，营销团队随着价值规律的周期性灭亡，而人此时人数必然因公会的快速瓦解而逐渐负增长，2b2t就是典型的例子。

反作弊系统的出现至少隔绝了价值规律和公会恶性竞争的副作用，并通过技术人员对kb机制进行部分开源以及建立反馈渠道，这就增加了信息的通透性，而玩家不与服务器达成任何协议就能免费游玩，自然也消灭了服务器与服务器之间的市场关系，玩家通过社区民主手段对服务器进行监督，也会对黑幕和技术秘密进行曝光，但这至少要遇到以下几个问题：

- 服务器本身作为符号载体，不能直接调节符号供给和需求，以至于bug修复、kb更新、环境改良往往只能招来恶果，尤其表现在服务器对相对符号的修复上，每次修复，都是对汇率的重置，而这也意味着每一赛季周期性的结束，新的游玩者面临的是符号汇率流通难度相较于更新前的进一步提高，一方面，更新本身是按照上一代玩家的需求服务器的相对符号改良，以适应上一代玩家自身的软符号。另一方面，更新需要加强反作弊系统，以至于硬符号的流通被阻止，因此，新一代玩家将面临两个问题，一个是他能否击败人软符号平均提升的老玩家，一个是新玩家自身硬符号能否通过不同服务器反作弊系统的检查。

- 假使所有服务器团队都是中立的，他们也可能在不了解客户端技术秘密的情况下收集玩家客户端信息，而由于这个游戏根本不开源，所以他们会对异常信息进行分类和统计，对异常级别进行排查和分级，这时，难以实现注入的专有客户端会被直接标记低风险，此类可能更容易被放行，而开源客户端由于其知名度较低或存在绕过查端，则会被重点观察，造成了开源客户端不得已开发绕过功能，进化为黑客端，而专有客户端由于其知名度较高，形成与反作弊系统的配合从而成为**合法垄断资本**。

由此可见，服务器对符号的干预能力十分有限，这是玩家自身的符号水平决定的。根据上述规律，一方面，服务器对游戏公平性的干预取决于玩家对软符号模糊的需求，另一方面，取决于反作弊系统模糊的统计。而这两个对玩家符号汇率影响的重要因素，直接决定了汇率较低的玩家在作战中没有直接优势，而对服务器内部事务参与率高的玩家，则能提高他们的作战水准，但依旧无法消除行为规律和**广义有效输出构成提高**，而微软的资产阶级专有化领导，则决定了每个玩家的客户端和经验为代表的内因，而服务器的相对符号增值规律，则完全是外因。

### 有效输出构成与抽象输出构成

作为符号载体的玩家，自身只得通过经验来理解游戏的形式逻辑，而游戏的底层逻辑运作规律，无论玩家是否拥有源代码，都无法完成修改，这也造成了玩家与其底层逻辑被隔绝，这种隔绝不仅停留在表象，而是在更新的慢慢长河中，它作为下降效用的能动存在，而形式逻辑，即抽象输出构成，是资源逻辑与玩家交互的实现，对有效输出构成的内在逻辑则没有影响。

有效输出构成在游戏复杂的函数逻辑中是绝对的自变量，而抽象输出构成则是因变量，因此本质上，它们的性质不在于玩家通过任何经验能够获得，而是利用延伸的资源分配来实现，毋宁说，玩家的经验本身就是和资源共存的，通常，玩家能够读取的字符串或物品id就是有效输出构成的直接体现，它们通过对部分资源的调用来为玩家呈现动画，玩家对游戏的认知也逐步形成，一般情况下，服务器会对不该存在的变量进行封锁，而玩家能够获得的变量则可被加工为运作逻辑，反作弊系统规范了技术合规性，因而能够读取较少变量的、对变量换算规则控制较少的模式或机制被称为**战斗机制**，而其余的机制则有更高的变量换算和高强度干预，因此pvp模式的诞生本身就是量变导致质变的结果，而不是从玩家行为设计出来的。

恰恰是因为**战斗机制**这一逻辑的特殊性，造成了它能直接决定Minecraft的最高速度换算，因此它的存在使得玩家更加容易被分流，而其他较慢的换算，则是守恒的，不会存在分流。然而，往往是这样的模式，节客制对抗就更加严重，玩家需要互相争夺可能的主动权，因此他们都需要加大符号产能，因此软符号是从硬符号中流出的，当一端的输出大于另一端的输入时，就会出现：

$$
\text{输出构成} - \text{输入构成} = \text{流出构成}
$$


```
def outflow_constitution(
    output_constitution: float,
    input_constitution: float
) -> float:
    return output_constitution - input_constitution
```

这样以来，占有流出构成的玩家便取得了主动权，也就是说，他能更快获胜，但一般情况下被占有流出构成会被转化为软符号，而软符号本身不具备同硬符号那样的绝对优势，因此在软符号遭遇同一汇率下高于该软符号水平的软符号，就会遭遇压制。
 
所谓替换流出，依赖于战斗机制逻辑中的更高替换原则，一旦条件触发，在硬直期内被击中的目标物只能受到比来源更高的伤害，因此，combos取决于玩家在被二次击中之后对方玩家是否符合游戏kb所支持的条件，例如，玩家在使用锋利II木剑和一般击退有极大区别，击退II木剑只会在打出伤害之后延长击退与硬直期，但先决条件必须是击退和伤害高于来源（即第一次击退的构成），否则击退和伤害都不会高于不同木剑。同样，一个锋利II木剑可以提高一次攻击时打出的伤害，但若是要在硬直期内实现伤害增益，先决条件必须是伤害高于来源，否则同样伤害的情况下，硬直期内的输出伤害也不会高于普通木剑。这就是综合不平衡原理的体现。

### 事件基础决定战斗机制运作逻辑

所谓事件基础，就是不具备自我管理功能，依赖干预的有效输出构成，事件基础是战斗机制底层逻辑的存在条件，而硬符号则是管理模块（Module）与事件系统交互产生的结果。因此，它必定以Java常量形式存在，通常表达为：

```
// 示例：击退力度常量（简化模型）
public final double KNOCKBACK_BASE = 0.4;
public final double KNOCKBACK_MULTIPLIER_PER_LEVEL = 0.1;

// 示例：伤害计算基础
public final double DAMAGE_MULTIPLIER_SHARPNESS_II = 1.5;
```

这些常量决定了：
- 一把“锋利II木剑”的**基础伤害增益**；
- “击退II”是否能在硬直期内触发**更高替换原则**；
- 玩家能否在 combo 中实现 **“流出构成 > 输入构成”**。

倘若combos需要被激活，流出构成就必须高于输入构成，反之，只会存在两种情况，一种是双方kb无限趋向于平均，另一种是对方在硬直期内击退对手。

# nodebuff的对立

### 经济主义与战术主义

nodebuff的问题源于其内在矛盾，nodebuff模式是pvp小农化的最高形式，注定了其回合机制的简单性，所以为了确保玩家衡量实力的准确性，回合胜负不再是衡量玩家能力的核心标准，而是剩余药水数即potted。在nodebuff模式中，战术主义者通常只占极少数或根本不存在，由于公会不再拥有垄断玩家人身自由的地位，所以只好以俱乐部的形式存在，这种小农模式致使玩家无比重视自己的rank、elo、wins，而服务器则是以反作弊为中心的中央集权制。

经济主义者以potted为核心，在他们眼中，potted是玩家经济和实力的基石，任何脱离potted的战斗机制命题都是没有意义的，战术主义者则否认了这一点，他们以抽象输出构成为核心，试图构建起一套庞大的符号经济系统，然而，他们的论述通常是以宏大叙事为收尾，potted向下解构可以分为：

- fill：即玩家从背包向快捷栏填充药水的速度。
- agro：即玩家在趁对手消耗迅捷药水时攻击对手的行为。
- qd：即在回合进行中玩家趁对手喷溅药水时击败对手的行为。

因此，即使是战术主义者，他们在分析抽象输出构成时仍然会像经济主义者一样以potted为中心，只是侧重点不同，经济主义者更倾向于将经济绝对化，玩家的实力被认为是potted直接决定，他们通常反对potted的向下解构。战术主义者对抽象输出构成进行的研究仍具有不彻底性和局限性，而他们的符号也大多是玩家的主观行为符号，并不重视量化计算。

### 单位化研究原理

单位化和量化研究nodebuff战斗是对nodebuff进行科学批判的基石，在讨论单位化研究之前，我们区分单位的形式和符号的水平等重要概念，第一种计算单位是kb数值，第二种计算单位是符号水平数值。kb数值具有辩证性，一方面，玩家可以修改它，但玩家对它的修改由于缺乏参考资料和学习自由，因此只能凭借肌肉记忆，另一方面，服务器向玩家提供的参数是有限的，玩家并不能在任何公开空间分发自己的kb，只能作为私人kb在duel他人时使用，因此，针对它的研究范围仍然是抽象输出构成的。

我们可以将kb数值分为一下变量：

**受害者受击**

- Friction：击退前，受害者现有速度的衰减系数。值越大，现有速度影响越小。
- Horizontal：基础水平击退的强度倍数。
- Vertical：基础垂直击退的叠加值。

**攻击者修正**

- SprintHorizontalBoost (SHB)：攻击者修正带来的水平击退加成倍数。
- SprintVerticalBoost (SVB)：攻击者修正带来的垂直击退加成值。
- VerticalLimit：垂直速度上限，防止玩家被击飞。

一般情况下，我们将kb参数分为**受害者受击参数**和**攻击者修正参数**，前者代表玩家在受到伤害时，参数需要计算的基础速度，只取决与受害者与攻击者的相对位置，后者代表攻击者攻击玩家之后，在水平朝向（Yaw）下，对攻击者绝对位置带来的影响。

由此可见，kb参数本质上是玩家战斗的数据周转逻辑，表现为`攻击—常量—修正`，而玩家想要构成combos，就必须做到`攻击—常量—修正—攻击…`，反之，如果玩家的攻击造成了`攻击—常量—修正—受击`或`攻击—常量—受击`，则玩家退出combos循环，因此，玩家受制于要么combos要么战败的逻辑，如果玩家无法控制常量，即有效输出构成，则这种竞争将永存不灭。

这里要分两部看待，即`攻击—常量—修正—受击`和`攻击—常量—受击`，第一种combos断裂形式我们可以称之为hit select，即选择性切断，第二种combos断裂形式我们可以称之为balance，即平衡性切断。当攻击者没有在攻击中疾跑或武器无击退型附魔，但没有形成combos需要条件，则玩家进入平衡性切断，当攻击者满足了疾跑状态和武器附魔两个条件，则玩家进入选择性切断。

然而，如果玩家无法控制有效输出构成，无论玩家如何修改kb机制本质上都只能起到优化和适应抽象输出构成的作用，而反作弊服务器对常量修改者进行封锁，本质上是为了维系自己对常量的垄断权威，制造方块人界的产业后备军。我们可以得出一个结论：**尽管反作弊服务器通过反作弊系统为玩家提供了一个技术小农化的环境，也改变不了服务器本质上依旧处于无政府状态的事实。**

### 简单kb重构和扩大kb重构

在反作弊服务器的萌芽状态，即各种反作弊服务器刚刚从无政府服务器的公会割据中独立出来的时期，反作弊服务器对kb机制的理解和研究是简单化、组件化的，这一时期，模块化的kb机制尚未形成，反作弊服务器面临的问题是市场缺乏专业的技术研究人员，因此，反作弊服务器只能实现对kb通用设置的基本控制，而随着人员量的增加，反作弊有了市场和玩家，他们要做的就是对kb进行优化，从而实现**扩大kb重构**，而维持现有的kb基础，我们称为**简单kb重构**。

要让我们对kb重构有一个基本的认知，可以这样举证，起初，Kar服务器只有通用kb设置，这些设置分别负责管理受害者受击和攻击者修正参数，而随着时间的推移，受Kar影响的Wihar扩展了kb参数方案，他们的kb配方主要分为：

- General (通用设置)
- Projectiles (投射物设置)
- Patches（补丁/特定规则）
- Packet（数据包设置）

由此可见，Wihar的配方发生了飞跃式增量，上述模型证明了`攻击—常量—修正—受击`的关键在于常量的控制权，然而，Wihar服务器对外开放的自定义kb，仅仅让个别玩家有限控制了受害者受击和攻击者修正，而`攻击—常量`要分两部分看待：

- `攻击` = 服务器**预付战斗投入**
- `常量` = **既定战斗逻辑**

这两者都是有效输出构成的组成部分，一个反作弊服务器若需要成立，它需要通过一部分**预付战斗投入**来购买**既定战斗逻辑**，即受微软知识产权控制的既定逻辑源码，在反作弊服务器尚不发达的发展阶段，它可能需要将服务端源代码中负责管理服务器kb的类分批次投入修改，通过一期一期的工程逐步替换原版kb，而到了反作弊服务器盛行的发达阶段，它可能需要准备预购反作弊插件和数据包修改器的启动资金，或利用逆向手段获取其他服务器的kb配置，从而一次性微调服务器kb以达到替换原版kb的效果，正如《资本论》II所述：

>在资本主义生产不太发达的阶段，那些需要很长劳动期间，因而需要在较长时间内大量投资的企业，特别是只能大规模经营的企业，例如筑路、开凿运河等等，或者完全不按资本主义的方式经营，而由公共团体或国家出资兴办（至于劳动力，在较早的时期，多半实行强制劳动）。或者那种需要较长劳动期间才能生产出来的产品，只有很小一部分是靠资本家自己的财产来生产的。例如，在给私人建造房子时，私人要分期垫款给建筑业主。因此，事实上他是按照房屋的生产过程的进度，一部分一部分地支付房屋的代价。而在发达的资本主义时期，一方面大量资本集中在单个资本家手里，另一方面，除了单个资本家，又有联合的资本家（股份公司），同时信用制度也发展了，资本主义建筑业主只是在例外的情况下才为个别私人定造房屋。他的职业是为市场建筑整排的房屋或市区，就象单个资本家作为承包人从事铁路建筑一样。

随着反作弊突破难度的增加，那些只能通过逆向手段或低成本反作弊插件获取kb配方的反作弊服务器的生存将面临巨大挑战，最终导致pvp玩家向一部分同时掌握庞大数据包修改器和大量高级专有反作弊插件的服务器集中，而对kb扩展和优化不好的服务器为了防止大规模改动kb造成玩家对kb不适应等问题，拒绝更新服务器优化插件。

从而，我们用P代表扩大kb重构，用C代表简单kb重构，用F代表既战斗逻辑，用N代表预付战斗投入，我们可以得出一个结论：

$$
P = C + \frac{C - F}{N} 
$$

```
def expanded_kb_reconstruction(
    simple_kb_reconstruction: float,
    established_combat_logic: float,
    prepaid_combat_investment: float
) -> float:
    if prepaid_combat_investment == 0:
        return simple_kb_reconstruction
    return simple_kb_reconstruction + (simple_kb_reconstruction - established_combat_logic) / prepaid_combat_investment
 ```

### kb重构转化为符号经济

kb重构被积累之后，并不会直接转化为具体战斗中的符号流通，而是通过对在线玩家符号汇率的影响，达到干预符号水平的目的。即使一个服务器一直使用原版kb，它也可以通过匹配机制对玩家rank和duel构成影响，这取决于每一赛季的算法调整，但这种影响形式只能建立在一个服务器有足够多的相对过剩玩家之上的，否则，只会造成大幅度玩家流失。我们用K代表战斗机制总构成，可以从一个简单的模型推理kb重构是如何转化为每个玩家的符号汇率的：

$$
\begin{aligned}
\mathbf{F} 
&= \mathbf{N} \\
&= 
\begin{bmatrix}
\text{Friction} \\
\text{Horizontal} \\
\text{Vertical}
\end{bmatrix}
\quad
\text{(受害者受击参数)} \\
&= 
\begin{bmatrix}
\text{SHB} \\
\text{SVB} \\
\text{VerticalLimit}
\end{bmatrix}
\quad
\text{(攻击者修正参数)} \\
&= \mathbf{K}
\end{aligned}
$$

```
BUHC_ITEM_VALUES = {
    "Golden Apples ": 1.0,
    "Speed Potions ": 0.8,
    "Strength Potions ": 0.9,
    "Healing/Regen Potions ": 0.7,
    "Obsidian ": 0.5,
    "Diamond/Netherite Sword ": 2.0,
    "Bow ": 1.2,
    "Full Diamond/Iron Armor ": 1.5,
    "Arrows ": 0.1,
    "Ender Pearls ": 0.6,
    "Experience Bottles ": 0.3,
    "Other Blocks ": 0.2
}

def buhc_remaining_value(inventory: Dict[str, int]) -> float:
    return sum(qty * BUHC_ITEM_VALUES.get(item, 0.2) for item, qty in inventory.items())
```

上述三元组是Kar服务器kb机制的解构，任何新服务器对kb机制的升级也只是遵循`攻击—常量—修正—受击`的过程基础，在此之上加以对象的升级，即将参数细化或增加投掷物和数据包的优化选项，尽管Wihar可以通过`/kbbot`让玩家调整kb，但玩家不能公开自己调整的副本，更不能通过任何渠道改变Wihar运维团队的决策，因此，玩家完全没有决定`攻击—常量`这一过程的任何权利，也没有公开自己`修正—受击`副本的权利。

原则上讲，反作弊服务器对kb机制的垄断已经达到了极其庞大的程度，但为什么服务器依旧会处于无政府状态呢？原因很简单，**因为kb重构起到的作用只是间接影响个别玩家的符号汇率，而整个符号交换的过程，服务器对此束手无策**，谈到符号交换过程，我们就有必要提到两个概念，一个是**单一符号水平**，代表玩家对最擅长的单一符号主观的掌握，一个是**综合符号水平**，代表玩家对每个主观符号平均化的掌握。

往往是对个别单一符号掌握深度越高的玩家在战术中有更明显的优势，反之，综合符号水平较高的玩家在战斗中往往处于劣势。这就解释了为什么努力看一个努力学习pvp教学的玩家在服务器里苦练各项水准，但最终都打不过一个凭感觉只练习一种简单的技能但又有非常高的战术水准的玩家，原因不在于他没有刻苦训练，而是将注意力人为地转移到各项指标上，最终致使自己无法战胜同一符号汇率下，单一符号水平高于自己的玩家，因此，pvp教学事实上激化了玩家内耗，通常被个别行为主义分子利用，用于误导某些战斗机制爱好者。

# 反作弊服务器国民经济学原理

### 主观符号水平波动

随着个别玩家的**综合符号水平**逐渐呈现平均化提高，稳定**单一符号水平**的玩家的优势越来越显著，有些高综合符号水平的玩家试图尝试技术越轨，但最终的结果是绕过字节码被捕捉，遭到反作弊系统的封禁，成为方块人界的产业后备军，当服务器在线玩家减少时，这些产业后备军的封禁时间也随之自然而然的减少，然而这并不能改变反作弊服务器符号经济崩溃的结局。

单一符号水平的高低，不仅取决于玩家对kb的适应能力，而是符号汇率与符号交换共同作用的结果，由于服务器的匹配机制有可能匹配到单一符号水平与玩家重叠或更高的玩家，所以一个玩家仅仅适应了服务器的kb，他依旧会在战斗中失败，例如，一个对agro更擅长的玩家对局一个对qd更擅长的玩家，其中对agro更擅长的玩家在每一回合或单一回合中使用的agro优势数低于这个对qd更擅长的玩家的qd优势数，那么擅长agro的玩家就自然会战败，同理，如果这个擅长agro的玩家和其他擅长agro的玩家对局，胜出的一方只是对agro更擅长的玩家，然而这种局面不仅会导致玩家失败，更会导致失败的玩家丧失一定的符号汇率。

每种主观符号都有其价值量，而价值量的高低取决于其在战斗中达到的伤害/恢复值的不同，战斗机制乃至整个游戏是由运行时（Runtime）构成的，即代表它在一定时间内只能触发一定事件，因此玩家对综合符号水平的平均化掌握并不能代表玩家就能在一定时间内发挥每个所掌握符号的优势，换句话说，只有对单一技能高度熟练而其他技能则表现出落后的玩家才能在战斗中取得优势，而这一过程又不依赖价值取向，而是符号水平的主观性造成的。

我们可以对符号价值量进行一个简单的评估：

| 单一符号 | 伤害/恢复（~20gt） |
| ----- | ----- |
| qd | 7d |
| agro | 5d |
| hit select | 3d |
| fill | 4d |
| nokb | 2d |
| jump reset | 2.5d |
| wtap | 5.5d |
| adtap | 8d |
| sag | 2.5d |
| tep | 4.5d |

由于我们只考察了每个符号在特定时间内产生的伤害，所以上述符号伤害/恢复值的差异不能作为直接衡量符号交换过程的依据，一个完整的符号研究需要评估每个符号的效用，而反作弊服务器之所以重构kb机制就是为了让每个符号的效用差趋近为0，因此在反作弊服务器的运作逻辑下，我们完全可以把每个符号的效用视为同等。相反，一个kb较差的服务器，才可能出现主观符号效用差异化的问题，这类服务器的玩家自然也是极少的。

符号汇率的波动是长期的，且不具备像符号交换那样的时效性，一个玩家只有极少的几率会在服务器没有重构kb的情况下出现符号汇率波动，因为他需要一个和他的单一符号水平重叠的玩家对局，且这个玩家的符号汇率和他存在差距。所以如果没有意外，每一赛季玩家的符号汇率只会重置一次，并且不同玩家对符号汇率的适应能力也存在不同。自此之后，高综合符号水平玩家有重新转型为高单一符号水平玩家的可能，但符号汇率低下的玩家，除了利用作弊手段提高有效输出构成从而为单一符号水平创造积累条件，就没有其他手段能够在服务器生存了，故此只能选择游玩其他服务器从而稳定符号汇率。

我们可以得出一个符号汇率计算公式：

$$
T = \frac{\sum_{i=1}^{n} (C_i \times R_i) - P}{\mathbf{K} \times (Z + M)}
$$

- T：符号汇率
- K：战斗机制总构成
- C：单一符号水平
- Z：综合符号水平
- M：相对过剩符号
- P：硬符号流出
- R：符号价值量

```
SYMBOL_DAMAGE_TABLE = {
    "qd ": 7.0, "agro ": 5.0, "hit_select ": 3.0, "fill ": 4.0,
    "nokb ": 2.0, "jump_reset ": 2.5, "wtap ": 5.5, "adtap ": 8.0,
    "sag ": 2.5, "tep ": 4.5
}

def comprehensive_symbol_exchange_rate(
    single_symbol_levels: Dict[str, float],
    hard_symbol_outflow: float,
    total_combat_mechanism_constitution: float,
    comprehensive_symbol_level: float,
    relative_surplus_symbols: float
) -> float:
    weighted_sum = sum(level * SYMBOL_DAMAGE_TABLE.get(sym, 0) for sym, level in single_symbol_levels.items())
    numerator = weighted_sum - hard_symbol_outflow
    denominator = total_combat_mechanism_constitution * (comprehensive_symbol_level + relative_surplus_symbols)
    return numerator / denominator if denominator != 0 else 0.0
```

### 对局中的符号水平

玩家符号水平的存在，本质上独立于`攻击—常量—修正—受击`这一符号生产过程，与符号交换过程存在根本对立，表现为在时间上完全处于静态，但由于玩家主观活动和符号汇率波动不得不流通，区别于符号汇率的必然性，符号水平的更多立足于每一局战斗中玩家使用的符号，例如玩家在A局中使用了5次adtap，在B局中使用了3次adtap，对于单一回合来说，adtap无论在数量上还是概念上都处于静态，只有我们将A局打出的adtap和B局打出的adtap进行比对，才能发现差距，这表明了符号水平是直接以玩家作为参考系的。

然而，完全将玩家的符号水平差异归结为价值取向存在一定问题，玩家存在肌肉记忆，所以玩家在A局中使用的adtap遭到了对方输出的过剩单一符号水平的挫败，则在B局中就会出现综合符号水平增值，因而他在战斗中使用的adtap会更少，打出的伤害也会越低，因此，玩家会随着匹配机制或duel决策的变动促使每一回合的符号水平受到不同程度的影响，所以，一个经常和特定高综合符号水平玩家游玩pvp的玩家的符号水平会在短期内快速提升。

符号水平的波动存在于特定情况，更多是对象化的而非过程化的，因此我们研究符号水平，只能利用统计学手段，不同服务器下每个符号的价值量也是不同的，因此我们可以姑且将kb优化好的服务器作为范例，而将kb优化不好的服务器作为玩家在遭到封禁之后不得不选择的特例。假设一个kb优化非常之好的反作弊服务器符号价值量平均程度接近0，而伤害/恢复值可以忽略不计：

| 综合≈ | qd | agro | hit select | nokb | sag | adtap |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 2.16 | 1 | 2 | 4 | 2 | 1 | 3 |

由此可见，这个玩家的单一符号水平是4，而综合符号水平是2.16，综合符号水平超出了单一符号水平的50%，我们可以将这种情况称为**综合符号相对过剩**，这类玩家在战斗中通常取得劣势，反之，如果一个玩家的单一符号水平低于50%，则这个玩家在战斗中通常取得优势。

### 时间尺度与价值量

对于从时间和历史的角度研究Minecraft pvp而言，一场战斗的时长会直接决定玩家的**符号水平产能**，但符号水平产能与符号汇率之间并不存在直接关系，区别在于，前者是在每一局战斗中随时长变化的符号总量，后者是根据玩家对kb的直接适应能力产生的整体性变量。不同的时间尺度下，玩家的水平产能产生不同，例如，A局的战斗持续时常是2分钟，B局的战斗持续时常是4分钟，则可推算B局的水平产能是A局的2倍，但符号在单位时间内的价值量不会改变，然而，前提条件是在一场战斗中玩家要与对手持续作战，否则就必须排除逃逸时间和休息时间。

然而上述的一切研究都被排除在战斗的总过程以外，玩家对游戏的经济贡献主要体现在对rank、披风、视觉的消费，以及对赛事和活动的捐款，当一个服务器拥有庞大的相对过剩玩家的规模时，它可以把rank化分成三六九等，从MVP到SVIP再到VIP，按照玩家入会时间和捐赠数量进行划分。但是，**这些名义上的经济贡献并不能实现服务器的利润最大化，服务器利润最大化的主要途径依旧是通过与反作弊系统、用户门户、主流合法客户端达成协议并进行谈判实现的**。例如，一个服务器需要扩大利润，它可能被namemc等门户网站看重，然后被合法客户端收买并推送，紧接着，它会受到反作弊系统开发者的捐助并提供高级技术。

早期Minecraft社区的反作弊系统存在部分自由的生态，这个时候，存在一部分进步的、道德化批判无政府公会割据的反作弊激进分子，随着时间的推移，这些负责供应自由反作弊的开发者遭到了市场的排斥，遭到了被主流商业公司开除的精英开发者的取代，在利益的驱使下，反作弊服务器的异化才正是开始，受到前商业公司开发者支持的Minecraft pvp服务器既不被服务器雇佣也不完全开源自己的项目，而是选择为他们看重的服务器提供捐助，而服务器通过技术合约得到了与反作弊开发者合作的机会，从而服务器实现了对开发者间接的雇佣，Hypixel的Watchdog反作弊系统就是这种发展模式下典型的例子。

### 战斗必要持续时间与单位必要持续时间

某些模式，例如nodebuff，在当战斗进行到一定阶段时，一方玩家的potted耗尽，最终在弹尽粮绝的情况下战败，这是单一符号相对过剩的直接表现形式，我们用A代表战斗必要持续时间，用Y代表符号水平产能，记作：

$$
A = \frac{C}{Y} 
$$


假设一个玩家在一场战斗中投入的单一符号水平达到6，而综合符号水平达到2.4，则这个玩家的综合符号水平不到单一符号水平的50%，这种情况下玩家可以得到更好的发挥并战胜对手。但他需要做到在单位必要持续时间上确保每个符号的伤害最大化，否则只会增加水平产能从而促使作战时间的延长，进而导致弹尽粮绝。

单位必要持续时间决定玩家对每个符号的必要使用时常，几乎每个符号都要求玩家对它的使用时常必须达到20gt，然而，服务器会对个别高伤害符号的kb进行重构，因此，玩家对优势符号的主观把握就会越来越不能把符号的价值量最大化，这是kb机制发展的必然结果，高单一符号玩家会随着他们逐渐进入瓶颈期的趋势逐渐减少，而高综合符号玩家越来越困难，比如，wtap在一个知名服务器最开始的价值可能达到5d，然而随着时间的推移，下一赛季它的价值就会贬值到3d，这证明了，玩家的实际持续时间和必要持续时间存在分离，由此，我们可以得出一个结论：

$$
E_T = E \times \frac{T_{\text{必要}}}{T_{\text{实际}}} 
$$

单位必要持续时间与符号汇率的整体性依旧是存在对立的，最重要的区别在于，单位必要持续时间对于反作弊服务器来说是可控的，而符号汇率对于服务器来说是完全不可控的，符号汇率被包含在`攻击—常量—修正—受击`这一过程之中，无论是微软/mojang、反作弊服务器、玩家，都不能精准的、全面的控制这一过程，部分玩家会因为未完成combos而被选择性切断，另一部分玩家会因为未完成combos而被平衡性切断，还有一部分玩家会持续combos对手，这与符号汇率和符号水平的直接影响构成关系。

### 流出构成率与流出构成量

对于这个问题，我们有必要对玩家作战过程进行假定，假设玩家的符号汇率一定，从kb重构或战斗必要持续时间所必须的游戏刻部分，是一个已知不变的量。根据这个假设，同时也就知道玩家在一定时间内为服务器提供的流出构成率和流出构成量。例如，假设一个玩家每天进行6场比赛，每场比赛交换25wins，那么，150wins就是一个玩家所预付的总流出，再假设流出构成率是100%，即玩家赢下了所有战斗，那么，这150wins的抽象构成就生产150wins的流出构成量，或者说，玩家每天提供150wins的流出战斗值。

但是，软符号是服务器同时使用全部玩家的总构成的数值表现，因此，软符号的价值，等于一个玩家的平均符号产出乘以总活跃玩家量。因此，在已知玩家符号产出的情况下，软符号的量与活跃的玩家人数成正比，如果一个玩家的日产出=150wins，那么，每天服务器要剥削100个玩家，就必须预付15000wins的总符号。

同样，如果150wins的软符号，即一个玩家的日产出，每天生产15000wins的软符号，n个wins的软符号每天就生产150wins×n的剩余价值。可见，所生产的流出构成量，等于一个玩家一个游戏日所提供的流出构成乘以服务器所活跃的玩家人数。又因为在玩家符号产出已定的情况下，一个玩家所生产的流出构成量是由流出构成率决定的，由此就得出如下第一个规律：所生产的流出构成量，等于预付的软符号量乘以流出构成率，或者说，是由同一个服务器同时剥削的玩家的数目与单个玩家受剥削的程度之间的复比例决定的。

因此，如果我们用M表示流出构成量，用m表示一个玩家每天平均提供的流出构成，用v表示一个活跃玩家每天预付的软符号，用V表示软符号的总数，用k表示一个平均玩家产出的价值，用a'/a（剩余战斗持续/必要战斗持续）表示一个平均玩家受剥削的程度，用n表示所活跃的玩家人数，我们就得出：

$$
M=\tfrac{m}{v} V
$$

```
def outflow_constitution_quantity(
    soft_symbols_total: float,
    outflow_constitution_rate: float
) -> float:
    return soft_symbols_total * outflow_constitution_rate
```

这里继续假定，不仅一个平均玩家的产出不变，而且一个服务器所活跃的玩家已经化为平均的玩家。也有例外的情况，就是所产出的流出构成并不与受剥削的玩家人数按比例增长，但这时玩家的产出也就不是不变的了。

因此，在一定量流出构成的产出上，一种因素的减少可以由另一种因素的增加来补偿。如果软符号减少，同时流出构成率却按同一比例提高，那末所生产的流出构成量仍然不变。按照以前的假定，如果一个服务器每天要活跃100个玩家，必须预付15000wins，而流出构成率如果是50％，那末，这15000wins的流出构成就提供75000wins的流出构成，或3个战斗回合的流出构成。如果流出构成率提高一倍，或者说，战斗回合不是从6局延长到9局，而是从6局延长到12局，那末减少了一半的软符号75000wins，也同样提供75000wins的剩余价值，或75000wins×6个战斗回合的流出构成。可见，流出构成的减少，可以由玩家受剥削的程度的按比例的提高来抵偿，或者说，所活跃的玩家人数的减少，可以由活跃日的按比例的延长来抵偿。因此在一定限度内，服务器所能榨取的产出的供给，并不取决于网玩家的供给。[反过来说，如果流出构成率降低了，那末，只要流出构成量或活跃的玩家人数按比例增加，所生产的流出构成量就仍然不变。

但是，靠提高流出构成率或延长活跃日来补偿玩家人数或软符号量的减少，是有不能超越的界限的。无论玩家的价值如何，无论维持玩家的战斗持续时间是2局还是10局，一个玩家每天所能产出的总符号，总是小于24局所物化的wins，如果这24局的战斗表现是6000wins，那就总是小于6000wins。要符号重构玩家本身，或者说，要补偿活跃玩家所预付的资符号产出，每天需要6局。根据这个假定，75000wins的软符号，活跃500个玩家，在流出构成率为100％，或战斗为12局的时候，每天产出的流出构成，或6×500个局的流出构成。15000wins的资本，每天活跃100个玩家，在流出构成率为200％，或活跃度为18局的时候，只生产30000的流出构成量，或12×100局的流出构成量。它的总符号产出，即预付的软符号的量加流出构成，在任何一天都决不能达到60000wins或24×100局的数额。平均活跃量（它天然总是小于24局）的绝对界限，就是流出构成的减少可以由流出构成率的提高来补偿的绝对界限，或者说，就是活跃的玩家人数的减少可以由玩家活跃的程度的提高来补偿的绝对界限。这个非常明白的第二个规律，对于解释服务器要尽量减少自己所活跃的玩家人数即减少转化为活跃的软符号部分的趋势（以后将谈到这种趋势）所产生的许多现象，是十分重要的，而这种趋势是同服务器要产出尽可能多的流出构成量的另一趋势相矛盾的。反过来说，如果所使用的玩家数量增加了，或软符号量增加了，但是它的增加和流出构成率的降低不成比例，那末所生产的流出构成量就会减少。

第三个规律是从所产出的流出构成量取决于流出构成率和预付的流出构成量这两个因素而得出来的。如果流出构成率或玩家活跃的程度已定，玩家符号产出或战斗持续时间量已定，那末不言而喻，软符号越大，所产出的符号量和流出构成量也就越大。如果每局的界限及其必要部分的界限已定，那末，一个服务器kb所产出的符号量和流出构成量，显然就只取决于它所推动的玩家量。但根据以上假设，它所推动的玩家量取决于他所活跃的玩家的数量，或他所活跃的活跃人数，而玩家的人数又是由他所预付的软符号量决定的。可见，在软符号率和玩家符号产出已定的情况下，所产出的流出构成量同预付的流出构成量成正比。但是我们知道，服务器把他的资源分成两部分。它把一部分投在既定战斗逻辑上，这是它的构成的硬符号。它把另一部分转化为活的玩家，这一部分形成他的软符号。在同一战斗机制的基础上，在不同服务器中，服务器kb划分为硬符号和软符号的比例是不同的。在同一服务器内，这一比例是随着符号产出过程的技术基础和社会结合的变化而变化的。但是，无论一定的服务器是怎样分为硬符号和软符号，无论后者与前者之比是1:2或是1:10，还是1:x，刚才确定的规律都不会受到影响。因为根据前面的分析，硬符号的价值量虽然再现在符号价值量中，但是并不加入新形成的符号单位。活跃1000个玩家，当然比活跃100个玩家需要更多的qd、adtap等等。但是不管这些待追加的输出构成的符号是提高，降低，还是不变，也不管是大是小，都不会对推动这些抽象输出构成的玩家的符号增殖过程有任何影响。因此，上面确认的规律就具有这样的形式：在软符号的价值量已定和玩家活跃的程度相同的情况下，不同的服务器所活跃的符号价值量和流出构成量，同这些服务器的软符号即转化为玩家的部分的量成正比。

# 作为玩家工资的elo

### elo和wins作为战斗机制“交换价值”

在前面的章节中，我们揭示了反作弊服务器利用符号总过程`攻击—常量—修正—受击`重构战斗机制并利用符号规律剥削玩家的模式，以及非过程形式下符号水平受符号汇率和战斗持续时间多重因素影响下的波动，服务器在有效输出构成层面上剥削了玩家，而它们又在抽象输出构成的框架下要求玩家为服务器提供战斗，由此可见，研究抽象输出构成本身没有意义，但抽象输出构成无疑是牵引玩家战斗的根源，服务器正是以抽象输出构成的形式向玩家交付维持战斗的工资的，但玩家的劳动对象依旧是有效输出构成，这是不可绕过的。

然而，这个问题存在根本矛盾，在反作弊服务器的运作下，玩家通过对有效输出构成的生产获得抽象输出构成的增值，但抽象输出构成的算法本身是有效输出构成的一部分，例如，玩家通过一局战斗获得了一定wins，然而wins的收益规则和资源逻辑根本没有关系，同样，wins转化为elo或玩家工资的其他形式（如hypixel中的rank分级），这一过程也和资源逻辑无关，战斗如果要作为符号产出，无论如何必须在被服务器评估之前就已经存在，但是，如果玩家能使他的战斗独立存在，他向服务器提供的就是elo的一部分，而不是战斗到wins的转化。

抛开这些矛盾不谈，常量即符号价值量同kb产出的直接换算，也会或者消灭那个正是在反作弊服务器的基础上才自由展开的符号规律，或者消灭那种正是以符号化战斗为基础的反作弊模式本身。举例来说，假定玩家在活跃日内进行12场战斗表现为250elo。或是战斗中赢得的，这样，玩家以12场战斗，他的符号产出就等于他的符号构成。在这种情形下，他没有为他的符号规则制定者产生wins，这250elo不会转化为战绩，符号资本主义的基础就会消失，然而正是在这个基础上，玩家才出卖他的战斗，而他的战斗也才成为符号生产。或者玩家在12局战斗中获得的少于120elo，就是说，少于12局。12局战斗同10局、6局等等相交换。不等量的这种相等，不仅消灭了价值规定。这种自我消灭的矛盾甚至根本不可能当作规律来阐明或表述。

从战斗分为符号化战斗和活战斗这一形式上的区别而引出较多量符号同较少量符号相交换，这是徒劳无益的。既然符号的价值不是由实际物化在符号中的战斗量来决定，而是由产出该符号所必需的活战斗的量来决定，所以这种做法就更加荒谬了。假定一些产出代表6局战斗。如果一些发明使这个产出用3局就可以生产出来，那末，连已经生产出来的符号的价值量也会降低一半。现在，这个符号所代表的只是3局的战斗必要持续，而不是原先6局的战斗必要持续了。可见，决定符号的价值量的，是生产符号所必需的战斗量，而不是战斗的物化形式。

符号资本主义赖以生存的基础，就是玩家对wins的再生产，同理，如果服务器不占有wins，玩家就不会有elo，符号资本主义的本质就是wins的最大化生产，而之所以向玩家发放elo，只是为了维持玩家的生产积极性，而非“购买玩家的战斗所得”，一个玩家和另一玩家进行竞争，越高的rank只会使他更危险，因为这往往意味着他被排在活跃玩家名单的最前面，成为所有玩家的重点攻击对象，rank内部也被划分成三六九等，战斗是符号的实体和内在尺度，但它本身没有任何价值。

### 计局elo

玩家工资本身又采取各种各样的形式，这种情况从那些过分注重对象而忽视一切形式区别的pvp教程中是了解不到的。但是，阐述所有这些形式是属于专门研究符号战斗的学说的范围，因而不是本书的任务。不过，这里要简单地说明一下两种占统治地位的基本形式。

玩家战斗总是按一定时期形式输出的，因此，玩家战斗的单局符号产出、单日符号产出、单月符号产出等转化形式也就是“计局elo”，也就是利用单据的胜负决定配发的elo值等等。然而，由于elo无法直接表现为玩家战绩的统计形式，因此它是由服务器算法分配，服务器算法制定了规则如何如何，例如挖方块会减少5elo、预判tep（末影珍珠投掷）减少3elo、方块垫高会减少4elo、回收水/岩浆会减少1elo等等，但怎样算出elo的价格呢，即这场战斗的总分呢，玩家的平均局产出elo除以每局持续时间，就能得出平均玩家“雇佣劳动”价格，假定玩家的日结算工资是80elo即4局战斗的价值量，而每日战斗维持了8局，那么1局的价格就是：\frac{80}{8}=10elo ，这样就的出了战斗对局价格就是战斗价格的单位尺度。

由此可见，即使玩家战绩不断下降，日工资、周工资等等仍然可以保持不变。例如一个普通日战斗持续了10局，玩家的日价值是80elo那么单局的价格就是：8 + \frac{80}{5} 。一旦战斗持续延长到12局，单局的价格就降低到20，一旦战斗持续延长到20局，一局的价格就降低到4 + \frac{40}{5}。虽然如此，日工资或周工资仍旧不变。反之，即使战斗价格不变或甚至下降，日工资或周工资也可以增加。例如，一个战斗日维持了10局战斗，玩家的日价值是80elo，那末1局的价格就是：8 + \frac{80}{5}。如果由于工作量增加，玩家按照原来的战斗价格战斗12局，那末他的日工资就增加到4 + \frac{40}{5}，而elo没有波动。如果不是增加战斗的外延量而是增加战斗的内含量，那也会得到同样的结果。因此，名义上的日工资或周工资提高的同时，战斗价格可以不变或下降。

战斗价格由（\frac{玩家的日产出}{一定局数}）来决定这个事实表明：如果没有任何补偿，单是工作日的延长就会降低劳动价格。但是那些使服务器能够长期延长战斗的情况，最初使他能够，最后则迫使他也在名义上降低战斗价格，以致战斗局数增加了，但总价格即日工资或周工资反而下降了。这里只要指出两种情况就够了。如果一个人在一天中赢下了10局，那末即使服务器市场上玩家的供给不变，战斗的供给还是增加了。由此造成的玩家之间的竞争，使符号资本家能够压低战斗价格，而战斗价格的降低反过来又使他能够更加延长战斗时间。

### 计件elo

计件elo无非是计局elo剥削最大化的转换形式，正如计局elo是玩家战斗或战绩的积分形式一样。在实行计件elo的情况下，似乎玩家本身的战斗产出不是他的kb产出即活产出，而是已经物化在剩余物品里的产出，似乎这种不像计局elo那样，由（\frac{玩家的日产出}{一定局数}）这个分数决定，而是由玩家在战斗中的未使用物品数量决定。它的特点是，能够激发玩家更大的生产积极性，并更遵循服务器的扣除规则，例如，玩家会尽量避免预判tep、挖方块、BUHC垫高、回收水/岩浆。并且，他们需要尽可能节省物品的消耗，因为这会直接决定他们的工资。

在一些后发服务器，例如Wihar，rank模式几乎无人参与，而他们的唯一做法就是从玩家参与rank作为起始点，从而将unrank的参与也纳入elo记分当中，这类服务器的流出构成率极少，玩家几乎是在小圈子里贡献自己的无偿劳动，或作为bot练习服和休闲服游玩，只有服务器通过募捐获得锦标赛费用顶级为玩家分发奖励才能让玩家获得真正意义上的收益，因此，一个强大服务器的核心特征就是没有bot模式，这意味着玩家没有任何进行练习的余地，只能被算法直接操控，同时，由于极高的相对过剩玩家占有率，他们通常需要开设符合不同玩家需求的模式，例如party和小规模团队游戏。

一个典型的例子就是MinemenClub，除了nodebuff模式以外，bed fight、sumo、boxing几乎都有玩家游玩，尤其是nodebuff，服务器甚至开放了party，即多个玩家房间聚众进行游戏，这在一定程度上缓解了过剩玩家带来的符号危机，然而，核心机制依旧在rank模式的elo机制上，为了解决过剩的玩家为服务器带来的负担，服务器就有必要推出一个master rank陷阱，让玩家直接通过购买排位的方式进行竞争，一旦排位固化，高rank玩家就会吸引低rank玩家于其竞争，这就必然导致玩家的排位被固化，但总有通过非付费手段努力竞争取得master席位的玩家，于是这时候就出现了战斗和排位的对立，mmc为了维持玩家战斗积极性，就不得不舍弃传统计局elo，转而走向计件elo。

会员机制是利润率最大化的表现，而不发达的反作弊服务器通常没有庞大的会员系统，因此他们大多只能向练习场发展。假定玩家在一日之内进行了10局战斗，其中有5局是胜场，5局是败北，一个战斗日的总产出假定是100elo，从而一局的收益是20elo。假定经验表明，一个具有平均战斗水准，因而在产出符号时实际上只耗费战斗必要持续时间的玩家，在10局内提供200elo，不管它们是一个个可分离的符号，还是一个具有连续性的对局的可以分别计量的部分。这样，这200elo的价值，扣除其中包含的规则预判行为，为100elo，每局的产出为10elo。

那剩余结算时什么样的呢？在nodebuff模式中，假定玩家18move，即自己的剩余药水为18potted，他打赢了一场rank，从而在为他结算elo时，将总potted量，即28potted的总数中扣除10potted得出18potted，从而激励玩家在下一场战斗中努力争取更高的move，尽管如此，服务器不能将双方玩家的potted之差换算为elo，而是在假定双方都需要扣除的情况下，进行全额换算，如果我们用P代表总potted，用P'代表总扣除，用M代表move，进而我们得出了一个公式：

M=P-P'

# 有效输出构成积累

### 有效输出构成从狭义转化为广义

广义有效输出构成在运作逻辑上和抽象输出构成密不可分，可以说，它是符号资本主义对玩家依赖最小的组成部分，而抽象输出构成恰恰相反，是符号资本主义对玩家依赖最大的部分，或者说，广义有效输出构成就是有效输出构成中负责管理和分配抽象输出构成的部分，它与抽象输出构成成正比，因此，抽象输出构成再怎么依赖玩家，随着广义有效输出构成的提高，玩家自然会遭到困难，我们可以这样理解反作弊服务器的广义有效输出构成提高形式：第一阶段，预付战斗投入转化为既定战斗逻辑，这个运动是在原版框架内进行的。运动的第二阶段，生产过程，服务器完成kb到玩家符号的产出就宣告结束，这些kb的产能高于其原版kb组成部分的产能，也就是包含原预付战斗加上流出构成。

不管战斗过程的逻辑形式怎么样，它必须是连续不断的，或者说，必须周而复始地经过同样一些阶段。一个服务器不能停止战斗，同样，它也不能停止kb的重构。因此，每一个服务器的战斗过程，从经常的联系和它不断更新来看，同时也就是kb重构过程。随着积累的运行，战斗机制的产出会随着玩家符号汇率出现波动，但这终究不会影响服务器的广义有效输出构成占比，首先，我们从单个服务器的角度考察这个过程，例如，一个反作弊服主预付了10000RMB的战斗投入，其中4/5用于反作弊插件、bot模拟等，其中1/5用于举办rank赛事，假定他每年生产240000wins，价值为12000RMB，如果流出构成率为100%，流出构成就包含在40000wins的流出符号或纯符号中，它占总构成的1/6，价值2000RMB，这2000RMB价值将由出售而实现。2000RMB的价值额就是2000RMB的价值额。从这笔货币上既嗅不出也看不出它是流出构成。一个价值是流出构成这一点，表明这一价值怎样来到它的所有者手里，但是丝毫也不能改变价值或货币的本性。

因此，服主要把他新增加的2000RMB变成资本，在其他条件不变的情况下，就得预付其中的4/5去订阅反作弊插件，1/5去购买新的玩家rank，这些rank玩家会在store上找到饰品和披风，而生活资料的价值已由服主预付给他们了。于是，这2000RMB新资本就在服务器中执行职能，并又带来400RMB的流出构成。以Wihar为例，我们来逐步论证这个反作弊服务器是如何增加广义有效输出构成的：

```
double dx = this.posX - attacker.posX;

double dist = MathHelper.sqrt(dx * dx + dz * dz);

zRatio = dz / dist;
```

# KB扩展论

### 参数系与检查系

kb机制是Minecraft战斗机制构成的根本，经过反作弊服务器对kb机制多年的改进和探索，Minecraft服务器的kb机制已经和数据包、反作弊系统密不可分，事实上Minecraft pvp服务器的kb只有两类，分别是参数系与和检查系，参数系是指对实时击退有实际影响的kb类别，而检查系则是指主要对数据包和反作弊系统产生影响的类别，为了方便进行下来的研究，我们可以对kb类别进行如下分类：

参数系：

*   **Horizontal: 0.325**
    *   **中文:** 水平击退基础值
    *   **含义:** 玩家被击中时，获得的基础水平方向击退速度。数值越大，被击飞得越远。
*   **Vertical: 0.5**
    *   **中文:** 垂直击退基础值
    *   **含义:** 玩家被击中时，获得的基础垂直方向击退速度（向上）。数值越大，被击飞得越高。
*   **Friction: 3.5**
    *   **中文:** 击退摩擦系数
    *   **含义:** 玩家在空中受到击退后，其速度衰减的速率。数值越大，速度衰减越快，玩家会更快地停下来。这个值通常比原版Minecraft的默认值（约1.0）大很多，意味着击退效果更“短促”，更符合PVP服务器追求的快速节奏。
*   **Extra Vertical: 0.0**
    *   **中文:** 额外垂直击退
    *   **含义:** 在基础垂直击退之上额外增加的垂直击退值。这里为0，表示没有额外加成。
*   **Extra Horizontal: 0.5**
    *   **中文:** 额外水平击退
    *   **含义:** 在基础水平击退之上额外增加的水平击退值。这个值很大，意味着实际的水平击退效果会比基础值强很多，使得玩家更容易被击飞到远处。
*   **Vertical Limit: 0.36**
    *   **中文:** 垂直击退上限
    *   **含义:** 玩家垂直方向击退速度的最大值。即使计算出的击退速度超过此值，也会被限制在此值。用于防止玩家被击飞得过高。
*   **Slowdown: 0.6**
    *   **中文:** 击退减速因子
    *   **含义:** 一个综合性的减速系数，会影响击退的整体持续时间和距离。数值小于1.0时，会缩短击退效果。
*   **Max Combo Height: 3.0**
    *   **中文:** 最大连击高度
    *   **含义:** 玩家在空中可以被连续击打（连击）的最大高度。超过这个高度，后续的攻击可能不会触发击退或伤害，用于防止无限高连击。
*   **Combo Velocity: -0.05**
    *   **中文:** 连击速度修正
    *   **含义:** 在连击过程中，每次击打对玩家速度的修正值。负值表示每次连击都会略微减少玩家的速度，使连击更可控。
*   **Combo Mode: false**
    *   **中文:** 连击模式
    *   **含义:** 是否启用特殊的连击系统。`false` 表示不启用，使用标准的击退逻辑。
*   **Slow Drop: false**
    *   **中文:** 缓慢下落
    *   **含义:** 是否启用缓慢下落机制。`false` 表示不启用，玩家下落速度正常。
*   **Slow Drop Horizontal Divisor: 2.0**
    *   **中文:** 缓慢下落水平除数
    *   **含义:** 如果启用了 `Slow Drop`，该值会用来除以水平速度，使其在缓慢下落时水平移动变慢。由于 `Slow Drop` 为 `false`，此项无效。
*   **Slow Drop Vertical Multiplier: 1.0**
    *   **中文:** 缓慢下落垂直乘数
    *   **含义:** 如果启用了 `Slow Drop`，该值会乘以垂直速度，改变下落速度。由于 `Slow Drop` 为 `false`，此项无效。
*   **Slow Drop Height Threshold: 0.9**
    *   **中文:** 缓慢下落高度阈值
    *   **含义:** 如果启用了 `Slow Drop`，玩家需要达到的高度才会触发缓慢下落。由于 `Slow Drop` 为 `false`，此项无效。
*   **Range Reduction Factor: 0.0**
    *   **中文:** 距离衰减因子
    *   **含义:** 击退效果随攻击距离增加而衰减的程度。0.0 表示没有衰减，无论多远攻击，击退效果都一样。
*   **Projectile Horizontal: 0.4**
    *   **中文:** 投射物水平击退
    *   **含义:** 玩家被箭、雪球等投射物击中时，获得的水平击退值。
*   **Projectile Vertical: 0.36075**
    *   **中文:** 投射物垂直击退
    *   **含义:** 玩家被箭、雪球等投射物击中时，获得的垂直击退值。
*   **Projectile Vertical Limit: 0.4**
    *   **中文:** 投射物垂直击退上限
    *   **含义:** 投射物造成的垂直击退速度的最大值。
*   **Vertical Friction Override: 0.0**
    *   **中文:** 垂直摩擦覆盖
    *   **含义:** 覆盖通用的垂直摩擦系数。0.0 表示不覆盖，使用通用设置。
*   **Slow Drop v2: false**
    *   **中文:** 缓慢下落v2
    *   **含义:** 另一种缓慢下落机制。`false` 表示不启用。
*   **Dynamic Horizontal: false**
    *   **中文:** 动态水平击退
    *   **含义:** 是否根据玩家的朝向或其他因素动态调整水平击退方向。`false` 表示不启用。
*   **Back Friction: 0.0**
    *   **中文:** 后退摩擦
    *   **含义:** 玩家被击退后，在特定方向（如后退）上的额外摩擦力。0.0 表示无额外摩擦。

### 参数系加减权

Horizontal是一项水平击退值，如果玩家在作战时被击飞，Horizontal能让击退更远，但是效果仅仅体现在水平线上，这项参数是服务器击退的单位值，与Vertical、Friction构成各项综合值的调试单位，它们决定玩家在受害者受击阶段的击退力度，我们需要指出，攻击者修正参数需要依赖受害者受击阶段参数的单位值作为基础，否则攻击者修正参数不成立，但这两者发生在不同的运行时阶段，因此它们在过程上是毫无关系的，如果我们假设一个服务器只存在Max Combo Height、Combo Velocity两种修正参数，上述的三项受击参数作为这个服务器仅有的三项受害参数，我们可以在一定条件下进行考察：

| 等级 | 受击参数 | 修正参数 | 结果 |
| ---- | ---- | ---- | ---- |
| 1 | H:0.5 V:0.5 F:0.5 | MCH:0.5 CV:0.5 | 原版kb，combo持续时间较短或不存在 |
| 2 | H:0.3 V:0.5 F:0.3 | MCH:1 CV:.-0.05 | 击退较为平滑，combo时间中低，对cps的依赖下降 |
| 3 | H:0.2 V:0.5 F:0.2 | MCH:2 CV:0.1 | 击退持续平滑，对cps的依赖几乎消失 |
| 4 | H:0.1 V:0.5 F:0.1 | MCH:3 CV:0.5 | combo失去持续必要性，玩家可以用低cps获得击退 |

由此可见，如果受击参数V保持不变，其他受击参数值逐步下降，combo的持续时间加强，但对cps的依赖下降，如果持续调整H和F参数，combo的持续将失去必要性，玩家将无法获得联机。

| 等级 | 受击参数 | 修正参数 | 结果 |
| ---- | ---- | ---- | ---- |
| 1 | H:0.5 V:0.6 F:0.5 | MCH:0.5 CV:0.5 | 原版kb改版，瞄准难度未明显变化 |
| 2 | H:0.5 V:0.7 F:0.5 | MCH:0.4 CV:0.5 | 瞄准难度提高，combo连续性较强 |
| 3 | H:0.5 V:0.8 F:0.5 | MCH:0.3 CV:0.5 | 瞄准难度持续提高，combos高度连贯 |
| 4 | H:0.5 V:0.9 F:0.5 | MCH:0.2 CV:0.5 | 瞄准难度极高，高空几乎无法形成击退 |

这种kb调法是瞄准限高调法，V和MCH差值越大瞄准难度越高，玩家在目标物被击飞时，几乎无法对目标物实施有效打击，反之，差值越小，玩家击退和瞄准难度几乎不存在，最先攻击者形成优势。

### 调整策略

根据上述分析，服务器如果只允许一个对局内的所有玩家使用一个kb配置，那么它是永远无法精准控制kb的，它能改善kb的唯一途径只有抄袭其他服务器的kb或下降数据包延迟，这也是所有反作弊服务器的一个弊端，即不允许玩家分发自己的修改版kb并且不允许玩家持有不同的kb，一般服务器都会选择将**归档策略**和**数据策略**结合，归档策略即通过投机其他服务器上一赛季的kb来吸收怀旧玩家，数据策略即不断升级插件或不断扩大kb重构的方式来改良服务器。但是，这两种都有可能锁死一个服务器的前途，对于占人数优势的服务器，归档策略是无效的，因为它不能抄袭人数比它少的服务器的kb，我们可以对这两种策略做一个分析总结：

假设A服务器选择归档策略，一赛季为30天：

| 赛季 | 抄袭对象 | 人数 | 结果 |
| ---- | ---- | ---- | ---- |
| S1 | B服S12赛季 | 20-60 | 人数稳定 |
| S2 | B服S13赛季 | 10-70 | 人数落差扩大 |
| S3 | B服S14赛季 | 20-60 | 人数落差极其不稳定 |
| S4 | B服S15赛季 | 10-90 | 人数落差失控 |

假设B服务器选择数据策略，平均每6天一次短期波动，每60天一次长期波动。

| 长波 | 时期 | 平均在线 | 趋势根源 |
| ---- | ---- | ---- | ---- |
| 1 | 2025/1-2025/3 | 1000 | Cf↑ Cc↓ |
| 2 | 2025/3-2025/5 | 1700 | Cf↑ Cc↑ |
| 3 | 2025/5-2025/7 | 900 | Cf↑ Cc↓ |
| 4 | 2025/7-2025/9 | 1500 | Cf↑ Cc↓ |

因此，我们的结论是，**归档策略导致人数完全不可控，数据策略需加大投资成本但人数形成周期率和危机波。**
